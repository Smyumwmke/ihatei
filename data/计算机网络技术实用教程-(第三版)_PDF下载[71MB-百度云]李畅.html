    (1)确定安装方式
    Windows 2000 Server提供两种安装方式，即升级方式与全新安装方式。所谓升级安装，就
是指在当前装有Windows NT 3．51 Server或Windows NT 4．0 Server操作系统的磁盘分区中，将
Windows 2000 Server自动安装在原有操作系统所在的文件夹中。如果用户计算机上安装的是
Windows NT 4．0 Server的企业版，则可以升级到Windows 2000 Advanced Server，而不是
Windows 2000 Server。如果用户计算机上安装了Windows NT 3．51 Server以前的服务器版本，
则不能直接升级到Windows 2000 Server，而必须首先升级到Windows NT 3．51 Server或
Windows NT 4．0 Server。
    所谓全新安装，是指在安装Windows 2000 Server的同时删除以前版本的操作系统，或在以
前没有安装操作系统的磁盘分区安装Windows 2000 Server。全新安装将导致以前操作系统中的
应用程序不能在Windows 2000 Server中使用，因此，必须在Windows 2000 Server中重新安装
这些应用程序。
    同全新安装相比，升级安装有许多优点，它不仅可以简化配置，如当前的用户账号、组和
用户权限都将被保存下来，而且不用在新的操作系统中安装以前的应用程序，但需要注意的是，
不能在压缩后的NTFS磁盘分区上进行Windows 2000 Server的升级安装。同样，全新安装也有
自己的优点，即可以通过在已经安装了操作系统的磁盘分区上安装Windows 2000 Server的方
法，使计算机具备双启动功能，利用双启动功能，可以保留以前操作系统的配置。
    (2)确定系统的引导方式
    利用Windows 2000 Server的全新安装方式，可以实现计算机的双启动。所谓双启动，是指
每次启动计算机的时候允许用户在几个操作系统中进行选择。例如，如果用户经常需要在
Windows 2000 Server和Windows NT Server 4．0之间进行转换，或者需要使用某些必须在某个特
定操作系统中运行的应用程序，则可以使用双启动模式。将系统设置为双启动后，每次启动计
算机时，会显示出所有可供选择的操作系统，如果用户不做选择，则启动默认的操作系统，当
然，默认操作系统可以由用户指定。选择双启动之前，应当注意以下几点：
    ①当Windows 2000 Server和Windows 9x上实现双启动时，若使用Windows 2000 Server，
则不能进行磁盘压缩操作。
    ②要实现MS-DOS、Windows 9x以及Windows 2000 Server之间的双启动，应当*后安装
Windows 2000，而且，硬盘上的主分区应当采用FAT或FAT32文件系统，而不是NTFS文件
系统。
    ③要实现Windows 2000 Server和其他操作系统之间的双启动，则应当将Windows 2000
Server安装在一个独立的分区，这样可以确保其他操作系统中的重要文件不被破坏。由于
Windows 2000 Server的NTFS文件格式有许多新的特点，所以使用了这些新特点的文件就只能
通过启动Windows 2000 Server来进行使用和阅读，例如在Windows 2000 Server中进行加密的
文件，在其他操作系统中则无法读取。
    ④Windows 2000 Server各个版本之间也可以实现双启动，此时，需要注意的是：在一台
计算机上安装Windows 2000 Server的不同版本时，必须使用不同的计算机名称，因为windows
2000 Server在安装时为域中的每台计算机产生一个唯一的安全标识(Security Identifier，SID)。
  (3)确定文件系统
  安装规划中的重要一步是确定Windows 2000 Server的安装位置及采用的文件系统结构。
Windows 2000 Server允许将系统格式化为FAT文件系统、FAT32文件系统或NTFS文件系统。
其中，FAT文件系统和FAT32文件系统基本上相同，只是FAT32文件系统支持更大的磁盘分区。
由于NTFS文件系统提供更加完善的安全设置，因此Windows 2000 Server的理想文件系统格式
应该为NTFS。当然，如果希望保存原来已经存储在FAT文件格式磁盘分区中的数据或实现双
启动，用户就应该在硬盘上用FAT或FAT32分区作为主分区(启动分区)。另外，FAT文件系
统和FAT32文件系统可以在Windows 2000 Server运行过程中随时更改为NTFS文件系统，但
NTFS文件系统却不能更改为FAT格式和FAT32格式，因此只有在用户确保NTFS为理想文件
系统时才采用NTFS格式。下面分别介绍这三种文件系统的特点：
    ①FAT文件格式。磁盘容量为1．2 MB(软磁盘的容量)到4 GB，支持的文件大小*大为
2 GB。FAT磁盘分区可以通过MS-DOS，各种版本的Windows、Windows NT和Windows 2000
Server，以及OS／2等操作系统进行访问。
    ②FAT32文件格式。磁盘容量为512 MB(软磁盘的容量)到2 TB，支持的文件大小*大
为4 GB。在Windows 2000 Server中，*多可以将FAT32文件系统分区格式化到32 GB。在本
地，可以通过各种版本的Windows 98和Windows 2000 Server操作系统访问。
    ③NTFS文件格式。*小的磁盘卷容量为10 MB左右，*大的磁盘容量为2 TB，不能在
软盘中使用，支持的文件大小只受磁盘卷容量限制，不受NTFS格式本身的限制。NTFS文件
分区在本地只能通过Windows 2000 Server操作系统或者安装了Service Pack 4以后版本的
Windows NT 4．0进行访问。
    2．Windows 2000 Server的活动目录
    (1)活动目录、目录、目录服务
    活动目录(Active Directory，AD)是Windows 2000 Server中的新增功能，它不但可以用
于存储服务器、文件、打印机以及网络用户与计算机账号等共享资源，实现普通意义上Windows
2000 Server目录的功能，而且它还是一个规则集模式，可以定义目录所包含的对象类与属性，
以及这些对象实例的限制及其名称格式等。活动目录还引进了一些新特色，例如，它是一个全
局编录，允许用户及管理者在不知道指定数据在目录中的具体位置的情况下，搜索到其详细信
息；它是一个排序与索引机制，允许网络用户与应用程序查询并发布对象及其属性；它是网络
中分布目录数据的一个复制服务，允许用户快捷、准确地完成复制操作；它集成了安全子系统，
存储域口令限制策略及访问控制等域安全策略，并实施基于对象的安全模型和访问控制机制，
增强了目录复制服务的安全性。
    目录是Windows操作系统中用于存储用户感兴趣对象的信息资源。其中，文件系统中的目
录用于存储文件信息，而在分布式计算机系统以及互联网等公共计算机网络中，目录除用于存
储文件信息外，还可以存储如打印机、传真服务器、应用程序、数据库以及用户等其他对象。
